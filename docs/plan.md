## EXECUTIVE SUMMARY

The MTI Scores API is a RESTful service that exposes Maritime Transportation Indicator (MTI) scores for vessels identified by their 7-digit IMO number. Consumers can retrieve the latest available scores, filter by year, or filter by a specific year-and-month combination. The API is built as a Spring Boot 3.2 application backed by PostgreSQL, following a layered architecture (controller → service → repository) with cross-cutting concerns handled by servlet filters and a global exception handler. Security is enforced via strict input validation, parameterized SQL queries, and token-bucket rate limiting (Bucket4j); all operations are traceable through a UUID request ID injected at the filter layer and propagated via SLF4J MDC.

### Integration Points

- **PostgreSQL 15**: Primary data store; table `mti_scores_history` holds all MTI score records. Connection configured via environment variables `DATABASE_URL`, `DATABASE_USERNAME`, `DATABASE_PASSWORD`.
- **Spring Actuator**: Exposes `/actuator/health` for liveness probes; configured to expose only the `health` endpoint.
- **Swagger UI**: Available at `/swagger-ui.html`; auto-generated by SpringDoc from controller annotations.
- No external message queues, caches, or third-party APIs are required.

### Technical Constraints

- IMO number must match regex `^[0-9]{7}$`; rejected with ERR_103 otherwise.
- Year must be in range 2000–2100; month must be in range 1–12; invalid values return ERR_104.
- Month without year returns ERR_102.
- Rate limit: 100 requests per minute per client IP (token-bucket, greedy refill).
- All score fields are NUMERIC(5,2) and may be NULL; NULL is serialized as `null` in JSON.
- All timestamps in ISO 8601 UTC format.
- Index `idx_mti_scores_imo_year_month` on `(imo_number, year DESC, month DESC)` required for query performance.

The backend implements a single REST endpoint (`GET /api/v1/vessels/{imo}/mti-scores`) backed by a PostgreSQL table `mti_scores_history`. Cross-cutting concerns — request ID tracing, rate limiting, error serialization — are handled by infrastructure classes that all business-logic classes depend on.

#### Story 4.1: MtiScoresRepository

**Background for implementer:** `JdbcTemplate.queryForObject` throws `EmptyResultDataAccessException` when zero rows match; this is caught and converted to `Optional.empty()`. JPA/Hibernate was deliberately avoided to keep queries explicit and prevent N+1 risks. The `RowMapper` must use `rs.getObject("created_at", OffsetDateTime.class)` (not `rs.getTimestamp`) to preserve the timezone offset stored in the `TIMESTAMP WITH TIME ZONE` column.

**Acceptance Criteria:**
- [ ] `MtiScoresRepository` is annotated `@Repository`
- [ ] `findLatest(String imoNumber)` executes `SELECT * FROM mti_scores_history WHERE imo_number = ? ORDER BY year DESC, month DESC LIMIT 1`
- [ ] `findLatestByYear(String imoNumber, int year)` executes `SELECT * FROM mti_scores_history WHERE imo_number = ? AND year = ? ORDER BY month DESC LIMIT 1`
- [ ] `findByYearAndMonth(String imoNumber, int year, int month)` executes `SELECT * FROM mti_scores_history WHERE imo_number = ? AND year = ? AND month = ? LIMIT 1`
- [ ] All three methods return `Optional<MtiScore>`; empty Optional when no row found
- [ ] `RowMapper` maps all 12 columns including `created_at` and `updated_at` using `rs.getObject(..., OffsetDateTime.class)`

### Backend API Contracts

```
GET /api/v1/vessels/{imo}/mti-scores

Path Parameters:
  imo       string    Required   7-digit IMO number; regex ^[0-9]{7}$

Query Parameters:
  year      integer   Optional   Filter by year; range 2000–2100
  month     integer   Optional   Filter by month; range 1–12; requires year

Request Headers:
  (none required)

Success Response — 200 OK:
  meta.request_id              string (UUID)
  meta.request_timestamp       string (ISO 8601 UTC)
  data.imo_number              string
  data.year                    integer
  data.month                   integer
  data.scores.mti_score        number | null
  data.scores.vessel_score     number | null
  data.scores.reporting_score  number | null
  data.scores.voyages_score    number | null
  data.scores.emissions_score  number | null
  data.scores.sanctions_score  number | null
  data.metadata.created_at     string (ISO 8601 UTC)
  data.metadata.updated_at     string (ISO 8601 UTC)

Response Header:
  X-Request-ID: <UUID>

Error Response — 4XX / 5XX:
  meta.request_id              string (UUID)
  meta.request_timestamp       string (ISO 8601 UTC)
  data.error_code              string
  data.title                   string
  data.message                 string

Error Code Reference:
  ERR_101   404   No scores found for given IMO/year/month
  ERR_102   400   Month specified without year
  ERR_103   400   IMO number must be 7 digits
```

### Backend Non-Functional Requirements

| Concern | Requirement |

| Rate Limiting | Token-bucket (Bucket4j); capacity 100, greedy refill 100/min; keyed by client IP; returns HTTP 429 |
| Testing | ≥80% line coverage; unit tests with Mockito; integration tests with Testcontainers PostgreSQL |
| Health / Docs | `/actuator/health` for liveness; `/swagger-ui.html` and `/v3/api-docs` for OpenAPI |

## RISK ASSESSMENT

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| PostgreSQL unavailable at startup | M | H | Flyway will fail fast with a clear error; use Docker Compose health check on `db` before starting `api` |
| Rate limit bypassed via IP spoofing | M | M | Current implementation keys on `RemoteAddr`; for production behind a proxy, read `X-Forwarded-For` header |
| NULL score fields cause NPE in mapping | L | H | `BigDecimal` fields in `MtiScore` are declared nullable; `rs.getBigDecimal` returns null for SQL NULL; Jackson serializes null BigDecimal as JSON `null` |
| Thread-pool IP bucket memory growth | L | M | `ConcurrentHashMap` grows unboundedly; for production consider eviction via Caffeine `Cache` with expiry |
| Bean Validation not triggered | L | H | `@Validated` on the controller class is required; `@Valid` alone does not fire for `@PathVariable` / `@RequestParam` |
